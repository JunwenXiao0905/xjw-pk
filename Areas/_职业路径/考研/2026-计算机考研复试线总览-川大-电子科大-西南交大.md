# 2026 计算机考研复试线总览：川大 / 电子科大 / 西南交大

## 这页准备回答什么
- `2026` 年三所成都高校里，计算机相关专业的复试线大概在什么区间
- 哪些学校是学校线，哪些已经细到学院线 / 专业线
- 从复试线角度看，三校的大致难度差异是什么

## 使用说明
- 这页先作为择校阶段的快速对照表
- 后续可以继续补：报录比、统考名额、调剂情况、复试占比、拟录取最低分
- 目前以官网公开复试线为准，个别学院若有二次划线，优先以学院通知为准

## 趋势解读
- `四川大学`：真正变难的是 `0854 电子信息`，而 `085405 软件工程` 在 `2025`、`2026` 被单独划线后，门槛显著低于同类热门方向。
- `电子科技大学`：学校线整体非常稳，`工学` 一直在 `310-315` 高位；`交叉学科` 和 `0854 电子信息` 都呈上升态势。
- `西南交通大学`：专业线波动最大，说明更适合按“具体专业”而不是按“学校整体难度”判断。
- 如果只看 `2026` 上岸性价比，当前表里相对友好的几条线主要是：`川大 085405`、`西南交大 081200`、`西南交大 085410`。

## 按初试科目横向对比

### 比较原则
- 横向比较不能只看学校和专业名称，优先看：`专业代码 → 初试科目 → 学院/方向 → 复试线`。
- 同样是 `计算机`，如果一个考 `408`，另一个考自命题，备考难度和分数线可比性都会变差。
- 同样是 `408`，还要继续看公共课组合：`英一/英二`、`数一/数二`。
- 同样是 `0854 电子信息`，不同学校可能拆成 `085404 计算机技术`、`085405 软件工程`、`085410 人工智能`，也可能仍按 `085400` 招生。

### 横向对比字段模板

| 字段 | 用法 |
| --- | --- |
| `学校` | 先标学校，不直接下结论 |
| `学院` | 同校不同学院可能线差很大 |
| `专业代码` | 判断是不是同一招生口径 |
| `专业名称 / 方向` | 处理 `085400` 这种大类招生 |
| `公共课` | 重点区分 `英一/英二`、`数一/数二` |
| `专业课` | 重点看是否 `408` |
| `2026 复试线` | 总分 + 单科线 |
| `可比性` | 标成 `强 / 中 / 弱` |
| `备注` | 写清楚非全、学院线、方向差异、是否待核对 |

### 暂时不要直接横比的情况
- `川大 085400 电子信息` 和 `成电 / 西南交大 085404 计算机技术`：方向相近，但专业代码不完全一致。
- `西南交大 信息科学与技术学院 085404 / 085410 / 085412` 和 `计智学院`：同校不同学院，可能考试科目、培养方向、复试内容都不同。
- `人工智能` 相关方向：可能分别落在 `1405`、`085410`、`085400` 或学院自设方向里，需要先拆目录。
- `软件工程`：虽然 `085405` 看起来最适合横比，但还必须核对全日制/非全日制、招生人数和是否单独划线。

### 当前优先比较顺序
1. 先比 `085405 软件工程`：三校都有明确软件工程口径，最适合做第一张横向表。
2. 再比 `081200 计算机科学与技术`：学硕主线清楚，但川大还需要补学院线。
3. 再比 `085404 计算机技术`：电子科大和西南交大可比性强，川大只能用 `085400` 相关方向粗比。
4. 最后比 `人工智能 / 智能科学`：代码和学院最复杂，适合单独做一页。

## 快速结论
- 只看学校整体门槛：`电子科大` 偏高
- 只看专业分化：`川大` 差异最大
- 只看择校时的可比性：`西南交大` 最直观
- 如果后面要继续做择校表，建议把每个目标专业拆成：
  - `学校`
  - `学院`
  - `专业代码`
  - `复试线`
  - `统招名额`
  - `复录比`
  - `拟录取最低分`
