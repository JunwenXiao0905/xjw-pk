# WebGPU 开源项目观察

## 当前判断
- 先把这类项目放在 `赛道观察` 里更合适，因为你现在还是在收集样本，不是在深度拆解单个仓库
- `wgpu` 更像底层基础设施，价值在于“判断 WebGPU 生态成熟度”和“看 Rust/WASM 这条技术链”
- `pascalorg/editor` 更像上层场景应用，价值在于“看 WebGPU 是否开始承载真实编辑器产品”
- 一个偏底层、一个偏应用层，组合起来看更有参考价值

## 项目 1：`wgpu`（暂按你的描述推断）

### 正式地址
- GitHub：<https://github.com/gfx-rs/wgpu>
- 官网 / 示例：<https://wgpu.rs/>

### 核验结果
- 仓库 README 明确写的是：`wgpu` 是一个 cross-platform、safe、pure-Rust graphics API
- README 同时明确写到：它在 `wasm` 上运行于 `WebGL2` 和 `WebGPU`
- GitHub 页面显示：截至 `2026-04-20`，`wgpu` 约 `16.9k stars`，最新 release 是 `v29.0.1`，发布日期为 `2026-03-26`

### 实际价值
- 这是 **基础设施级项目**，不是单一 demo
- 它的价值主要在：
  - 帮你判断 `Rust + WebGPU + WASM` 这条链是否足够成熟
  - 能作为跨平台图形层的观察样本，覆盖 native 与 web
  - 适合看底层渲染抽象、跨平台兼容、图形 API 工程化能力
- 但它的价值 **不在业务场景表达**，所以如果你的目标是“找可以直接借鉴的 3D 产品形态”，它不如上层编辑器直观

### 我的结论
- 值得长期跟踪
- 适合作为 `WebGPU 基础设施代表项目`
- 如果你后面要做深度拆解，可以单独迁到 `Projects/OpenSource/wgpu/`

## 项目 2：`pascalorg/editor`

### 正式地址
- GitHub：<https://github.com/pascalorg/editor>
- 在线产品：<https://editor.pascal.app/>
- Changelog：<https://editor.pascal.app/changelog>

### 核验结果
- GitHub README 明确写的是：`A 3D building editor built with React Three Fiber and WebGPU`
- 仓库 About 区直接挂了线上地址：`editor.pascal.app`
- GitHub 页面显示：截至 `2026-04-20`，仓库约 `13.5k stars`，最新 release 是 `v0.5.1`，发布日期为 `2026-04-14`
- 官方 changelog 页面显示：`2026-04-09` 发布了 `v1.0.0`，并强调它是 “A 3D editor built for humans and agents”，核心点是 `semantic scene graph`

### 实际价值
- 这是 **高价值样本**
- 原因不是单纯因为它用了 `WebGPU`，而是它已经把 `WebGPU` 放进了一个明确场景：`3D 建筑编辑器`
- 它值得看的点：
  - 不是渲染 demo，而是编辑器产品形态
  - 明显带有 `AI / agent-friendly` 设计倾向
  - 结合了 `React Three Fiber + WebGPU + 建筑/空间编辑`，说明 WebGPU 已经开始承接更复杂交互
  - 如果你在观察 `空间智能 / 建筑编辑 / AI 原生编辑器`，它比普通 3D engine demo 更有信号价值

### 我的结论
- 这个项目的观察优先级高于普通 WebGPU demo
- 它更像“赛道信号”，不是单纯技术玩具
- 如果后面你要拆它的架构、交互、数据模型，再迁到 `Projects/OpenSource/pascal-editor/`
